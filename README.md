
# Wireshark HTTP Traffic Analysis & Threat Detection

## 📌 Overview

This project demonstrates hands-on network traffic analysis using Wireshark in a simulated SOC environment. The focus was on analyzing HTTP traffic, identifying unusual patterns, and detecting potential security threats from captured PCAP file.

## 🎯 Objectives

* Analyze HTTP traffic from PCAP files
* Identify anomalous or suspicious request patterns
* Use Wireshark filters to isolate relevant traffic
* Reconstruct communication sessions

## 🛠️ Tools & Technologies

* Wireshark
* TCP/IP Protocol Suite
* HTTP Protocol
* Packet Analysis Techniques

## 🔍 Key Activities

* Applied display filters (e.g., "http", "http.response.code", "http.request.method") to isolate traffic
* Inspected HTTP requests and responses for anomalies
* Identified unusual URIs and suspicious request patterns
* Followed TCP streams to reconstruct sessions
* Investigated potential indicators of compromise (IOCs)

## 🚨 Key Findings

* Detected irregular HTTP request behavior indicating possible data exfiltration.
* Observed unusual request patterns that deviated from normal traffic behavior
* Identified suspicious endpoints during analysis

## 📊 Project Outcome

This project strengthened my ability to analyze packet-level data, detect anomalies, and simulate real-world SOC analyst workflows for incident investigation.
